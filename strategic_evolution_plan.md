# ACE Framework 戦略的進化ロードマップ (2025-2026)

## 1. はじめに
本ドキュメントは、最新のAgentic AIおよびRAGの研究動向（2025〜2026年予測）に基づき、ACE (Agentic Context Engineering) Frameworkを次世代の「コンテキスト・エンジン」へと進化させるための戦略的計画をまとめたものです。

現行の「同期対話ループ」と「非同期学習ループ」の分離という強みを活かしつつ、信頼性と汎用性を極限まで高めるための施策を定義します。

## 2. 成功要因（Success Factors）の深化
現在のアーキテクチャが持つ以下の強みを、さらに強化します。

- **非同期構造化学習 (Async MFR)**:
    推論と学習を分離する「Microservices Moment」的アプローチを継続。LLMの推論コストを最適化しつつ、バックグラウンドでの知識純化を推進。
- **インテント駆動型コンテキスト・エンジニアリング**:
    `Curator` による意図解釈を精緻化し、単なる検索ではない「文脈の動的生成」を高度化。

## 3. 潜在的リスクへの防御戦略 (Risk Mitigation)

### 3.1. メモリ汚染 (Memory Poisoning) への対策
ハルシネーションを含む誤った知識が長期記憶（LTM）に蓄積されるリスクを排除します。
- **Critic / Judge ノードの導入**:
    `Background Worker` の最終工程に検証ステップを追加。別の強力なモデル（Judge）が、抽出された知識の論理的整合性と既存知識との矛盾をチェックします。
- **エビデンス・スコアリング**:
    保存される各知識に「信頼スコア」を付与。複数回の対話で裏付けられた知識ほど、検索時に優先される仕組みを構築。

### 3.2. 検索粒度の最適化と関係性の保持
ベクトル検索の限界（点と点の推論の弱さ）を克服します。
- **構造化メモリ (Graph-Lite) への移行**:
    SQLiteのスキーマを拡張し、`(エンティティ) - [関係] -> (戦略/教訓)` というトリプレット構造を導入。
- **ハイブリッド・プランニング**:
    ベクトル検索で「似た状況」を探し、グラフ構造で「関連する依存関係や制約」を引き出す、多角的な検索アルゴリズムを実装。

### 3.3. コンテキスト・スタッフィングの抑制
「Lost in the Middle」現象を防ぎ、LLMの集中力を維持します。
- **Adaptive Context Injection**:
    検索された全ドキュメントを注入するのではなく、現在の回答に必須の要素のみを選別・要約して注入。

## 4. 次世代機能の戦略的ロードマップ

### Phase 1: Self-Reflection & Verifiability (即時着手)
- `Background Worker` への **Self-RAG** 的評価ロジックの統合。
- 知識保存時の「批判的検証プロンプト」の標準化。

### Phase 2: Adaptive Retrieval (適応的検索)
- `Curator` 冒頭への軽量な **Router** 実装。
- クエリの複雑性（挨拶、事実確認、深層推論）に応じた、検索実行の有無と強度の動的制御。

### Phase 3: Structural Graph Integration
- LTMにおけるエンティティ間の関係性定義の自動化。
- **RAS (Retrieval And Structuring)** 思想に基づく、検索結果の構造化要約提示。

## 5. 結論
ACE Frameworkは、単なるRAGから、ユーザーと共に成長する「自己改善型知能」へと進化するポテンシャルを持っています。本ロードマップに従い、「知識の量」よりも「知識の質と構造」に焦点を当てた開発を推進することで、2025年以降のAgentic AI市場において圧倒的な信頼性を持つフレームワークを目指します。
