# 🤝 10. コントリビューションガイド

ACE Frameworkへの貢献に興味を持っていただきありがとうございます！

## 開発環境のセットアップ

1. **フォークとクローン**: このリポジトリをフォークし、ローカルにクローンします。
2. **依存関係のインストール**: `uv sync` を実行して、開発用パッケージを含む全ての依存関係をインストールします。
3. **プレコミット・チェック**: コードをコミットする前に、静的解析を実行してください。

```bash
# Lintチェック (Ruff)
uv run ruff check .

# 型チェック (Mypy)
uv run mypy src
```

## 貢献の手順

### 1. プロンプトの改善
新しい言語のサポートや、既存のプロンプトの微調整を行う場合は、`src/ace_rm/prompts/` 内のファイルを編集してください。
- 共通のキー名（例: `UNIFIED_ANALYSIS_PROMPT`）を維持するように注意してください。

### 2. 新機能の追加 (エージェント・ノード)
エージェントの思考プロセスを変更したい場合は、`src/ace_rm/agent/graph.py` を編集します。
- 新しいノードを追加し、`StateGraph` に登録します。
- `AgentState` に新しいフィールドが必要な場合は、定義を更新してください。

### 3. 記憶バックエンドの変更
ベクトルデータベースの変更などを検討している場合は、`src/ace_rm/memory/core.py` のインターフェースを維持したまま、新しい実装クラスを作成することを推奨します。

## プルリクエスト (PR) のガイドライン
- **意味のあるコミットメッセージ**: 変更内容を簡潔に説明してください。
- **テストの追加**: 新機能やバグ修正には、必ず対応するテスト（`tests/`）を追加してください。
- **ドキュメントの更新**: Wikiの内容に変更が必要な場合は、併せて修正をお願いします。
